# Weak block-identifiability theorem for a simplified DDLP-like sequential model

## Setting

We consider a sequential object-centric generative model with three latent blocks:

- a sequence-level background variable $b_j \in \mathcal B$,
- object-level content variables $c_{j,1},\dots,c_{j,K} \in \mathcal C$,
- time-varying object positions $p_{j,i,t} \in \mathbb R^2$.

For each sequence $j$ and frame $t$, the observation is generated as

$$
x_{j,t} \,=\, B(b_j) \, + \, \sum_{i=1}^K T_{p_{j,i,t}}\bigl(D(c_{j,i})\bigr),
$$

where:

- $B : \mathcal B \to \mathcal X$ is the background decoder,
- $D : \mathcal C \to \mathcal X$ is the canonical centered object decoder,
- $T_p$ is a known translation operator acting on centered object templates.

The intended interpretation is that $D(c_{j,i})$ produces an object in canonical coordinates, while the explicit translation layer $T_{p_{j,i,t}}$ places it at the appropriate image location.

## Minimal assumption list

We assume the following.

### (A1) Sequence-wise constancy

For each sequence $j$,

$$
b_j \text{ is constant in } t,
\qquad
c_{j,i} \text{ is constant in } t \text{ for each } i.
$$

Thus only the positions $p_{j,i,t}$ evolve over time.

### (A2) Decoder injectivity on support

The maps $B$ and $D$ are injective on the support of the latent distributions:

$$
B(b)=B(b') \implies b=b',
\qquad
D(c)=D(c') \implies c=c'.
$$

This rules out background or content aliasing at the canonical-image level.

### (A3) Trivial stabilizers / absence of pathological symmetries

If

$$
T_u\bigl(D(c)\bigr) = T_v\bigl(D(c')\bigr)
$$

almost everywhere on the image domain, then necessarily

$$
u=v,
\qquad
c=c'.
$$

Equivalently, the only latent transformation that leaves a rendered canonical object unchanged is the identity. This excludes hidden symmetries that would allow position and content to trade off without changing the image.

### (A4) Object separation

For each frame, the translated object supports are pairwise disjoint:

$$
\operatorname{supp}\!\left(T_{p_{j,i,t}}D(c_{j,i})\right)
\cap
\operatorname{supp}\!\left(T_{p_{j,i',t}}D(c_{j,i'})\right)
=\varnothing
\quad \text{for } i\neq i'.
$$

This is a strong but clean assumption that avoids occlusion and overlap ambiguities.

### (A5) Dataset richness / anti-confounding condition

The data contain enough variation to decouple background from content. Formally, backgrounds recur with different content tuples and content tuples recur with different backgrounds. A sufficient version is:

for any admissible background $b$ there exist at least two sequences $(j,j')$ such that

$$
b_j=b_{j'}=b,
\qquad
(c_{j,1},\dots,c_{j,K}) \neq (c_{j',1},\dots,c_{j',K}),
$$

and for any admissible content tuple there exist at least two sequences sharing that tuple but with different backgrounds.

This prevents perfect confounding between sequence-constant foreground information and background information.

### (A6) Learner factorization with explicit recentering

The learned decoder has the same factorized form

$$
\hat x_{j,t}
= \hat B(\hat b_{j,t}) + \sum_{i=1}^K T_{\hat p_{j,i,t}}\bigl(\hat D(\hat c_{j,i,t})\bigr),
$$

with the important architectural constraint that $\hat D$ receives only content latents and not position latents. Position acts only through the explicit translation layer $T_{\hat p}$.

### (A7) Exact reconstruction and background-shuffle invariance at optimum

At an optimum, reconstruction is exact and the model is invariant to within-sequence shuffling of the learned background code:

$$
\hat x_{j,1:T}(\hat b_{j,1:T},\hat c,\hat p)
=
\hat x_{j,1:T}(\hat b_{j,\pi(1:T)},\hat c,\hat p)
\qquad \forall \pi \in S_T.
$$

This expresses that the learned background block is sequence-static.

## Theorem statement

## Theorem (weak block identifiability)

Assume (A1)--(A7). Let the data be generated by

$$
x_{j,t} \,=\, B(b_j) \, + \, \sum_{i=1}^K T_{p_{j,i,t}}\bigl(D(c_{j,i})\bigr),
$$

and let a learned model achieve exact reconstruction with the factorization in (A6) and invariance in (A7).

Then for every optimal solution there exist:

- an object permutation $\sigma \in S_K$,
- an invertible map $\phi_b : \mathcal B \to \hat{\mathcal B}$,
- invertible maps $\phi_i : \mathcal C \to \hat{\mathcal C}_i$,

such that for all sequences $j$, objects $i$, and times $t$,

$$
\hat b_j = \phi_b(b_j),
$$

$$
\hat c_{j,i} = \phi_i\bigl(c_{j,\sigma(i)}\bigr),
$$

and either exact or approximate positional recovery holds:

$$
\hat p_{j,i,t} = p_{j,\sigma(i),t},
$$

or, in the finite-resolution / approximate-decoder regime,

$$
\lVert \hat p_{j,i,t} - p_{j,\sigma(i),t} \rVert \le \varepsilon,
$$

for some architecture-dependent localization error $\varepsilon \ge 0$.

Hence:

- the background block is identifiable up to invertible reparameterization,
- the object-content blocks are identifiable up to object permutation and invertible reparameterization,
- the position blocks are identifiable up to the same permutation and possibly a bounded localization error.

## Why this theorem is weak rather than full identifiability

This theorem does **not** claim exact recovery of the original latent coordinates. It allows:

- invertible reparameterization of the background code,
- invertible reparameterization of each content code,
- permutation of object indices,
- bounded positional error in the approximate setting.

This is therefore a **block-identifiability** result rather than a full coordinate-identifiability result.

## Technical guidelines for a proof

A workable proof strategy would likely proceed in the following steps.

### Step 1: show sequence-level constancy of the learned background block

Use the background-shuffle invariance condition to show that the learned background code cannot depend on frame-specific information. At optimum, replacing $\hat b_{j,t}$ by any permutation within the same sequence leaves the reconstruction unchanged, so the reconstruction-relevant information in $\hat b$ must be sequence-level and frame-invariant.

### Step 2: show that the learned background block cannot absorb foreground content

This is the key anti-leakage step. By dataset richness, the same background appears with different object contents, and the same object contents appear with different backgrounds. If the learned background block encoded foreground content, then the learned background decoder would need to distinguish sequences with the same true background but different true contents, contradicting either injectivity or exact reconstruction under recombination. This yields that $\hat b$ depends only on $b$, up to an invertible map.

### Step 3: subtract background and isolate object contributions

Once the background block is identified up to reparameterization, compare

$$
x_{j,t} - B(b_j)
= \sum_{i=1}^K T_{p_{j,i,t}}\bigl(D(c_{j,i})\bigr)
$$

with the learned decomposition

$$
\hat x_{j,t} - \hat B(\hat b_j)
= \sum_{i=1}^K T_{\hat p_{j,i,t}}\bigl(\hat D(\hat c_{j,i})\bigr).
$$

The explicit recentering architecture guarantees that position enters only through the translation operator, which prevents positional leakage into the content branch.

### Step 4: use object separation and trivial stabilizers to obtain uniqueness up to permutation

Because the translated supports are disjoint, the sum decomposition is unique up to reordering. By (A3), equality of translated object images implies equality of both content and translation. This identifies each learned object branch with one ground-truth object branch up to permutation.

### Step 5: recover positions

After matching each learned object branch to a ground-truth object branch, exact translation structure yields exact position recovery in the idealized setting. In a more realistic finite-resolution model, one should expect a bounded localization error statement instead.

## What this theorem does not imply

Even if the blocks $(b, c_{1:K}, p_{1:K,1:T})$ are identifiable in the above sense, this does **not** imply that the latent transition law for the positions is identifiable. The reason is that any blockwise invertible reparameterization changes the representation of the transition law by conjugation. Thus one identifies the latent blocks only up to a gauge, and the transition law is determined only up to that same gauge unless additional assumptions are imposed.
